Chapter 8. How to use sendMessage.

If you want to communicate data between domains, use sendMessage via eventPage.

domainA ---> eventPage ---> domainB

Note that the scope of localStorage cannot be used for data communication between domains because it is a domain unit.

1. Prepare the necessary files.

The contents of the files are described on this Chapter.
Create the icon file with an application such as paint.

You can also download from the following Chrome Extension.
https://chrome.google.com/webstore/detail/how-to-make-chrome-extens/kcekgmndcdbeflkloloihblbepjnpfom?hl=en

2. Place the files to any folder on your PC.
Ex. C:\\ChromeExtenson\\Chapter8

3. Enter "chrome://extensions" in the chrome address bar. Enable Developer mode.

4. Click the "Load unpacked" to specify the folder.

5. Click the icon added to the right of the address bar.
Domain A and Domain B are displayed. These are for testing purposes and are actually local content within the extension.

6. Select domain A and click the Send button.


7. Select domain B and click the Recv button. Make sure that domain A data has been received.

domainA.js
document.getElementById('id_Send').onclick = function() {
  var value = document.getElementById('ta1').value;
  chrome.runtime.sendMessage({method: 'Send', key: 'key', value: value}, function (response) {
    ;
  });
};

domainB.js
document.getElementById('id_Recv').onclick = function() {
  chrome.runtime.sendMessage({method: 'Recv', key: 'key'}, function (response) {
    document.getElementById('tb1').value = response.value;
  });
};

event.js
var value;
chrome.runtime.onMessage.addListener(function (request, sender, sendResponse) {
  switch (request.method) {
  case 'Send':
    value = request.value;
    break;
  case 'Recv':
    sendResponse({value: value});
    break;
  }
});
